# 文档分类与保存（简化版）

**只保留真正值得留档的**，其余不落盘或临时放对话里。

---

## 放哪里

- **核心**：项目根目录 `core.md`、`memory.md`、`ads-performance.md`（不随意增删）
- **其他值得保留的 doc**：一律 **`docs/`**，不分子文件夹，用**文件名**区分即可

---

## 分类（三类）

| 类型 | 放什么 | 命名示例 |
|------|--------|----------|
| **参考** | 竞品、价格、市场等供 core/Agent 引用的数据 | `competitors-reference.md`、`SBD-price-€5499.md` |
| **审阅** | Coach/内部审阅、复盘、反馈 | `coach-review-b4-teaser.md` |
| **产出** | 要留档的邮件、新闻稿、提案等 | `buzz-newsletter-0305.md`、`street-BD-车店名.md` |

不需要的别建：如 setup、archive 等可等真有需要再说。

---

## 命名

- 简短、能看出是什么即可；同一主题多版可加日期（如 `0305`）
- 不必强制前缀，保持可读

---

## 给 Agent

生成 doc 且**值得保留**时，存 **`docs/`**，文件名简短、能看出类型和主题即可。临时产出可不落盘。

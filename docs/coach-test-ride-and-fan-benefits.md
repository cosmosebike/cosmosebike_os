# Coach：试驾落地性与粉丝福利怎么设置

**用途**：协调「我们真的该主推试驾吗」「落地性如何」「粉丝福利到底怎么设」，避免 Dealer/Street/Buzz 各做各的。**结论与规则由 Mia 拍板**；Coach 整理选项与落地性，供决策。

---

## 1. 我们真的应该用试驾吗？

### 支持主推试驾的理由

- **core 与冷启动顺序**：线下先行 → Deruiz 摆车 → 真实试驾 → 第一批口碑；试驾是冷启动第一环。
- **高客单价**：€5,199 的车，很多人要骑过才肯下单；试驾能直接抬转化。
- **与博主谈**：「你们粉丝可以来我们这里预约试驾、还有福利」——有具体动作，比纯「来看看我们网站」强。
- **信任**：Mönchengladbach 实体点 + 试驾，比纯线上可信。

### 试驾的落地性（现实约束）

| 约束 | 说明 |
|------|------|
| **只有一个点** | 目前只有 Deruiz（Mönchengladbach）；粉丝分散在德语区，很多人**到店成本高**，试驾人数天然有上限。 |
| **谁接待** | 试驾到店后谁接待、谁讲解、谁跟进？若全靠 Deruiz 店员，需与他们明确流程与话术（Street 协调）。 |
| **预约与履约** | 若用 Calendly：需有人同步 Deruiz 日历、避免约满或无人接待。若用表单：需有人转成预约并通知 Deruiz。**自动化只到「收到线索」**，到店履约要靠人。 |
| **规模** | 试驾适合**高意向、能到店**的人；不适合「先广撒网」。所以：**试驾要推，但别指望靠试驾吃下所有流量**；更多人是 Waitlist/留资，后续再推试驾或直购。 |

### Coach 建议（供 Mia 拍板）

- **试驾要保留**，作为「高意向、能到店」的闭环动作；首屏与博主话术里都可以提「Mönchengladbach 可试驾」。
- **落地性**：在推试驾前，**Street 与 Deruiz 确认**：开放时间、谁接待、单日可接待量、预约如何同步（Calendly 或人工）；D4 只负责站上表单/链接与确认邮件，**到店环节必须有人**。
- **话术分寸**：对**远距离粉丝**（如奥地利、瑞士）可弱化「必须试驾」，强调 Waitlist/预订 + 「若到德国可顺道试驾」；对**德国尤其北威州**可强推试驾。

## 2. 粉丝福利到底怎么设置

### 目标

- 博主愿意在视频/贴文里说「我粉丝去预约/预订有福利」；
- 我们可归因（谁从哪个博主来的）、可控制成本、不引发纠纷。

### 选项（供 Mia 选或组合）

| 方案 | 内容 | 优点 | 风险/成本 |
|------|------|------|------------|
| **A. 试驾即福利** | 「通过 [博主] 链接预约试驾 → 到店即送小礼品（如骑行小帽、水壶）」 | 简单、好说、促进到店 | 需备小礼品；到店才兑现，远距离粉丝拿不到 |
| **B. 折扣码** | 「通过 [博主] 链接进入 → 留资/Waitlist 或下单时填专属码 → 减 €50/€100」 | 可线上兑现、易归因 | 直接让利；需在 Shopify 设折扣码、D4 做码与 utm_source 对应 |
| **C. 优先/专属感** | 「通过 [博主] 链接 → 优先排期试驾 / 优先排 Waitlist / 专属客服」 | 几乎零成本 | 若产能或接待跟不上，「优先」会失信 |
| **D. 礼品+码二选一** | 到店试驾送小礼品；不到店则留资后送折扣码（如 €30） | 兼顾到店与不到店 | 规则稍复杂，需在站上与邮件里写清 |

### 落地性

- **归因**：所有福利链接带 **utm_source=博主名**（如 felix）；D4 在 CRM 中按 utm_source 打 tag，便于统计与和博主对账。
- **兑现**：若折扣码 → Shopify 折扣码由执行层建，规则（满减、仅首单等）由 Mia/Coach 定。若小礼品 → Street 与 Deruiz 确认谁备货、谁在试驾时发放。
- **文案**：福利规则一句话写清（如「通过 Felix 链接预约试驾并到店，可获得 XX」）；Buzz 与博主沟通时用同一句，Dealer 定稿，福利页放同一句，避免口径不一。

### Coach 建议（供 Mia 拍板）

- **先选一种主推**：例如「试驾即送小礼品」（A）或「留资/Waitlist 填博主码减 €30」（B），不要同时上三四种规则。
- **规则写进 core 或本 doc**：确定后由 Coach 更新 core 或本文「粉丝福利规则」一节，Buzz 与博主谈、D4 做页与码时都按这一版执行。
- **执行层实现**：福利页（德语/英语）、折扣码（若选 B/D）、表单「从哪得知」或 utm 打 tag 由懂 Shopify/Klaviyo 的人在 Dealer 需求下完成；规则不定不擅自发明。

| 事项 | 结论（待 Mia 定） | 谁落地 |
